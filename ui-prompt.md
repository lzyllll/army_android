# 4399 军队管理 App 原型设计需求

## 设计目标

设计一款手机端 App，用于管理多个 4399 游戏账号的军队/公会系统。

---

## 页面结构

### 1. 账号管理页（首页）

**顶部导航栏**
- 标题："军队管理"
- 右侧：添加账号按钮（+）

**账号列表**
- 卡片式展示每个账号
- 卡片内容：
  - 账号标识（用户名/UID）
  - 存档数量徽章
  - 展开/收起箭头
- 展开后显示该账号下的存档列表：
  - 存档项：索引 | 玩家名(等级) | 最后登录 | 作弊标记(红色)
  - 右侧开关：启用/禁用该存档
- 卡片底部操作：
  - 编辑 | 删除

**添加账号流程**
1. 点击 + 按钮
2. 弹出登录表单：
   - 输入用户名（4399账号）
   - 输入密码
   - 验证码输入框 + 验证码图片（点击刷新）
   - 登录按钮
3. 登录成功后自动获取 UID 和 Cookies
4. 自动获取存档列表并显示


**添加账号存档**
1. 点击账号存档
2. 选择已添加的账号
3. 根据已选择的账号存档，获取存档列表即  - 存档项：索引 | 玩家名(等级) | 最后登录 | 作弊标记(红色)
4. 添加

**底部 Tab 栏**
- 账号 | 军队 | 贡献 | 申请

---

### 2. 军队信息页

**顶部**
- 返回按钮
- 标题：当前选中的 GameUser 名称

**切换 GameUser**
- 下拉选择器：选择要查看的 GameUser

**军队信息卡片**
- 军队名称
- 军队等级
- 成员数量 / 上限
- 战力
- 公告（可展开）

**成员列表**
- 表头：玩家名 | 等级 | 战力 |日贡献| 周贡献|上周贡献|上上周贡献| 
- 成员行：
  - 头像占位
  - 玩家名
  - 等级
  - 战力
  - 日贡献
  - 周贡献，上周贡献，上上周贡献
  - 角色标签（会长/副会长/成员）
- 点击成员行：进入成员详情

**自身信息卡片**（悬浮底部或顶部）
- 玩家名 | 等级 | 个人贡献 |日贡献 |周贡献| 上周贡献|上上周贡献|

---

### 3. 贡献管理页

**顶部**
- 标题："贡献管理"
- 副标题：显示可贡献的 GameUser 数量

**批量操作区**
- "全选" 复选框
- "一键贡献" 按钮（醒目主色调）


**贡献详情弹窗**（点击一键贡献后）
- 标题："确认贡献"
- 列表：将要贡献的 GameUser 及金额
- 总计贡献金额
- 确认 | 取消 按钮
- 刷新贡献记录

---

### 4. 申请审核页

**顶部**
- 标题："申请审核"
- 下拉选择：选择 GameUser

**申请列表**
- 空状态：显示 "暂无申请"
- 申请项卡片：
  - 申请人名称
  - 等级 | 战力
  - VIP 等级
  - 申请时间
  - 操作按钮：
    - 通过（绿色）
    - 拒绝（红色/灰色）

**批量审核**
- 全选复选框
- 批量通过按钮
- 批量拒绝按钮

## 交互细节

### 账号添加流程
```
[+] → [登录弹窗] → [输入用户名/密码/验证码] → [LoginApi.login()] → [获取UID和Cookies] → [自动获取存档] → [显示在列表]
```


### 一键贡献流程
```
[选择GameUser] → [点击一键贡献] → [确认弹窗] → [确认] → [逐个执行贡献API] → [显示进度] → [完成Toast]
```

### 审核流程
```
[查看申请列表] → [点击通过/拒绝] → [API调用] → [从列表移除该项] → [Toast反馈]
```

## 视觉风格
简洁，白色但优雅

## 技术要求

- 响应式设计，适配主流手机屏幕
- 支持下拉刷新
- 支持离线缓存（账号信息本地存储）
- 错误处理友好提示
- 验证码图片支持点击刷新
