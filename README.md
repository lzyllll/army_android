# UI 现代化与重构总结

## 1. 设计理念
- **风格**: 现代、简洁、专业的仪表盘界面
- **主题**: 支持深色/浅色模式，注重高对比度和可读性。
- **组件**: 使用 `MaterialCardView`、圆角、细腻的阴影以及统一的调色板（蓝色/绿色/橙色表示状态）。

## 2. 关键模块重构

### A. 军团信息 (`UnionInfoFragment`)
- **新布局**:
  - **头部卡片**: 在现代渐变/纯色背景上突出显示军团名称、等级和 ID。
  - **数据网格**: 双列网格，在整洁的卡片中显示成员数量、排名和其他关键指标。
  - **操作区**: 简化的常用操作按钮。
- **目标**: 提供军团状态的即时、高层级概览。

### B. 个人信息 (`SelfInfoFragment`)
- **新布局**:
  - **个人资料卡**: 以流畅的水平布局显示用户头像、名称、ID 和等级。
  - **仪表盘网格**: 全面的网格布局，展示个人统计数据（贡献、战力等）。
  - **深色主题**: 针对视觉冲击力强的深色仪表盘外观进行了优化。

### C. 成员列表 (`UnionMembersFragment`)
- **新布局**:
  - **表格格式**: 用结构化的表格替换了简单的列表，带有表头以便更好地浏览数据。
  - **行设计**: 交替的行颜色（如适用）或清晰的分隔线以提高可读性。
- **功能**:
  - **管理员**: 用徽章清晰区分。
  - **状态**: 在线/离线状态指示器。

### D. 任务贡献 (`AccountTasksFragment`)
- **新布局**:
  - **头部**: 现代化的摘要头部。
  - 这里的贡献日志为绝对精准
  - **任务卡片**: 每个任务都作为一个独特的 `MaterialCardView` 展示，包含：
    - **图标**: 代表任务的图标。
    - **标题与进度**: 清晰的文字和线性进度条。
    - **操作按钮**: 具有状态感知的 "Go/领取" 按钮样式。
  - **进度条**: 自定义 Drawable (`bg_progress_bar.xml`) 实现圆角、平滑的进度指示。

### E. 捐献追踪 (`DonateFragment`)
- **重构**: 从 API 轮询转变为 **本地持久化**。
- **特性**:
  - **本地状态**: 捐献状态现在基于 "一键" 操作在本地进行追踪。但在军队中的贡献日志，为调用服务器查看，
    - **绿色 (已完成)**: 如果用户今天点击了 "一键"，则标记为完成。
    - **灰色 (未贡献)**: 新一天的默认状态。
  - **智能选择**: "一键" 操作现在尊重用户通过复选框进行的选择。
  - **防重复**: 智能跳过今天已经贡献过的账号。
  - 贡献查看一种是通过extra，一种是通过taskstatus
  - extra中为 游戏平台所添加的日志，还有存档里，也有个日志，taskstuas为服务器日志最为准确。造成无法贡献的原因就是因为extra，ui里任务你没做任务，但服务器其实已经上传了，不需要担心卡贡。
  - 由于登录账号时，会覆盖extra，所以我未对两个路径进行日志上传，贡献会造成卡贡假象，但你的总的贡献肯定会加上去的。
  - 贡献记录是可以伪造的，即本周数据，上周，上上周和日贡

## 3. 技术改进
- **数据持久化**: `AccountManager` 现在保存/加载 `lastDonateTime` 以支持本地追踪。
- **优化刷新**: 通过依赖本地状态来获取捐献状态，减少了不必要的 API 调用。
- **崩溃修复**: 解决了 Fragment 逻辑中的 `InflateException` 和各种空指针问题。

## 4. 功能逻辑说明

### 多账号一键贡献
- **逻辑**: 该功能是**客户端自动化**操作。
  1. 遍历当前在 `DonateFragment` 中选中的本地账号。
  2. **本地检查**: 检查每个账号的 `lastDonateTime`（本地记录）。如果判断今日已贡献，则**跳过**。
  3. **执行请求**: 对未贡献的账号，逐个发送贡献请求给服务器。
  4. **记录状态**: 请求成功后，更新本地的 `lastDonateTime` 为当前时间，界面显示为“已完成”。
- **特点**: 依赖本地时间戳，有效防止重复请求

### 军团贡献日志 (4399)
- **来源**: 数据直接通过 API (`getUnionLog`) 从 4399 游戏服务器获取。
- **内容**: 显示军团内**所有成员**（包括非本机账号）的真实贡献记录。
- **区别**:
  - **一键贡献**: 是**“去执行”**的操作，关注的是本机的账号是否完成了任务。
  - **贡献日志**: 是**“看结果”**的数据，关注的是军团整体的活跃度和历史记录（服务器权威数据）。
